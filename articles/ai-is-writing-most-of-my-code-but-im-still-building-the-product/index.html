AI Is Writing Most of My Code, But I'm Still Building The Product

A reflection on using AI heavily while keeping ownership of product direction, taste, debugging, and shipping decisions.

Ava Thalheim Jun 5, 2026 2 min read

The honest version

The honest version

AI is writing most of my code these days.

That does not make the product automatic.

I did not type a prompt and receive an app.

Most of the work ended up being product decisions.

Questions like:

  • Which features actually matter?
  • What should the UI look like?
  • What should the UI feel like?
  • How should purchases work?
  • Which features belong in Pro?
  • How should data be stored?
  • What architecture do I want?
  • How scalable does this need to be?
  • How should different platforms behave?
  • What is the smallest version of the product that still provides value?

Those questions don't disappear because AI can write code.

If anything, they become more important.

The implementation becomes faster, which means more time is spent thinking about the product itself.

When building Flowtime, AI helped me create screens, controllers, data models, and integrations much faster than I could have on my own.

But AI did not decide that the app should be local-first.

It did not decide to remove account creation.

It did not decide which features should be free and which belong in Pro.

It did not decide how the experience should feel when someone opens the app for the first time.

Those decisions are the product.

The code is only one part of software development.

There is design.

There is architecture.

There is testing.

There is distribution.

There is pricing.

There is support.

There is marketing.

There is understanding the problem you're trying to solve in the first place.

AI accelerated implementation, but it didn't remove the need for product design.

If anything, it allows me to be a better developer.

I can reach an MVP faster.

I can test ideas sooner.

I can spend more time thinking about the questions that actually matter.

Because software is more than code.

And building a product is more than implementing features.
